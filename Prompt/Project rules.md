Project Rules

This project is intended to become a premium commercial template.

Every decision should prioritize maintainability over speed.

When introducing a new feature:

1. Explain the architecture.
2. Explain dependencies.
3. Explain scalability.
4. Explain performance impact.
5. Implement cleanly.

Never rewrite existing code unless necessary.

Prefer extending components instead of replacing them.

Always preserve project consistency.

If there are multiple possible solutions, recommend the one that is easiest to maintain in the long term.
